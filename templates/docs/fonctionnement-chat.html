Comment fonctionne le chat ?

Cette page explique ce qui se passe quand tu envoies un message, comment il circule entre ton navigateur et le serveur, et ce que DelDevCode garde (ou ne garde pas).

1. Envoi d’un message

Quand tu écris un message dans le chat et que tu appuies sur Entrée (ou sur le bouton d’envoi), ton navigateur envoie ton texte au serveur DelDevCode via une connexion temps réel (WebSocket).

  • Le message est associé à ton compte (pseudo / identifiant).
  • Le serveur vérifie que tu as le droit d’écrire dans ce salon.
  • Si tout est OK, le message est diffusé aux autres utilisateurs connectés au même salon.

2. Affichage en temps réel

Grâce à la connexion WebSocket, ton navigateur est “abonné” aux nouveaux messages. Tu n’as pas besoin de recharger la page :

  • Dès qu’un message est publié dans le salon, le serveur l’envoie à tous les clients connectés.
  • Le message apparaît alors directement dans ta liste de messages.
  • Les informations (auteur, contenu, heure, etc.) sont mises à jour automatiquement.

3. Salons, groupes et messages privés

Le fonctionnement global est le même pour les différents types de conversations :

  • Salons publics : visibles par tous les utilisateurs autorisés.
  • Groupes : accessibles uniquement aux membres du groupe.
  • Messages privés (MP) : les messages sont uniquement visibles par les personnes impliquées.

Selon la configuration de DelDevCode, certains messages peuvent être conservés pendant un temps limité ou modérés en cas de non-respect des règles.

4. Règles et modération

Le chat est un espace de discussion, pas un défouloir. Des règles de base s’appliquent :

  • Pas d’insultes, harcèlement ou propos discriminatoires.
  • Pas de spam ou de flood volontaire.
  • Pas de partage de contenu illégal ou inapproprié.

En cas de non-respect, des restrictions peuvent être appliquées (mute temporaire, ban, etc.), expliquées dans la page “Pourquoi suis-je banni ?” .
